# Wiki Taiwan GIS: A Codex Agent Demo Site

This project is a demo website showing how to use only a Codex agent and a single prompt, or task instruction, to automatically complete data organization, JSON generation, a D3.js tree diagram, and a Leaflet + OpenStreetMap GIS visualization website.

This output website is a sample generated by Codex from the original prompt. It preserves the generated result as-is, without manual rewriting of the content, data, or interface.

The task in this example asks Codex to visit the Wikipedia page for "Administrative divisions of Taiwan", read the tables and descriptions on that page, and organize the administrative division changes in Taiwan from 1661 to 1895. Whenever an administrative division changed, Codex separated it into a new historical stage and created both an administrative hierarchy tree diagram and a GIS map for each stage.

This output website is based on content organized from Wikipedia and is for reference only. The main purpose of this project is to demonstrate a method: users can replace the text source, data topic, visualization type, and publishing method, then let a Codex agent generate a different structured dataset and interactive website from a new task prompt.

## Project Contents

This static website includes:

- `data/admin_periods.json`: data for Taiwan administrative division changes from 1661 to 1895.
- `index.html`: the static website entry point.
- `app.js`: D3.js tree diagram, Leaflet map, period switching, and GIS coordinate loading logic.
- `styles.css`: website layout and visual styling.
- `.nojekyll`: allows GitHub Pages to publish the static files directly.

Website features include:

- Timeline data segmented by administrative division changes.
- Administrative hierarchy tree diagram for each stage.
- GIS map for each stage.
- D3.js tree diagram rendering.
- Leaflet + OpenStreetMap map rendering.
- Automatic GIS coordinate lookup through the Nominatim/OpenStreetMap API.
- Built-in approximate fallback coordinates when API lookup fails.

## Usage

This is a pure static website. No npm packages are required.

Start a local server in the project directory:

```bash
python3 -m http.server 8000
```

Open the website in your browser:

```txt
http://localhost:8000/
```

Opening `index.html` directly from the file manager is not recommended, because the browser may block `fetch("./data/admin_periods.json")` from reading a local JSON file.

## Recreate This with Codex

1. Create a new empty folder.
2. Initialize a git repo in that folder.
3. Open Codex agent.
4. Paste the task instruction from "Prompt for Codex".
5. Let Codex read the Wikipedia page, organize the data, and create the JSON file and static website.
6. Check whether the generated `data/admin_periods.json`, tree diagram, and GIS map meet your requirements.
7. Preview the website with a local server.
8. Push the project to GitHub and publish it with GitHub Pages.

For a detailed step-by-step Codex workflow, see:

## Notes

- This output website is based on content organized from Wikipedia and is for reference only.
- This project is an educational demo and output example, not an official historical database.
- The main purpose of this project is to demonstrate a method. Users can replace the text source, data topic, visualization method, and website presentation format.
- GIS coordinates are approximate points based on modern place names or historical administrative seats.
- The actual boundaries of historical administrative divisions may differ from modern administrative boundaries.
- If the Nominatim API lookup fails, the website uses built-in approximate coordinates.
- If you need to query geographic coordinates at scale, follow the Nominatim usage policy and avoid high-frequency requests.
